### Patterns and Recurring Elements:

*   **Consistent Timestamping**: All entries are from `14/1/2026` or `17/1/2026`, indicating focused development during these two days.
*   **Targeted Platform Development**: The project primarily targets `webos` (LG) and `tizen` (Samsung) smart TV platforms, along with `hbbtv` and `browser` for development/testing.
*   **Build Automation (`package-launcher-apps.cjs`)**: This file is central to the build process, managing:
    *   **CLI Argument Parsing**: Always supports `--platform` (`-p`) and `--env` (`-e`).
    *   **Environment Support**: Consistently validates `DEV`, `PRE`, and `PRO` environments.
    *   **Temporary Directories**: Creation and recursive removal of temporary build directories (`tmp_webos_...`, `tmp_tizen_...`) for clean builds.
    *   **Package Naming Convention**: Standardized output filenames like `appName_PLATFORM_YYYYMMDD_vVERSION_ENV.ext`.
    *   **External CLI Tools**: Reliance on `ares-package` for WebOS and `tizen package` for Tizen.
*   **Version Management**:
    *   Initially used `pkgVersion` from `package.json`.
    *   Evolved to interactive prompts (`readline`) for `binVersion` (semantic versioning `X.X.X`) to provide more control during packaging.
    *   Version injection into platform-specific configuration files (`appinfo.json` for WebOS, `config.xml` for Tizen) is a key recurring task.
*   **Platform-Specific Logic**:
    *   **WebOS**: Consistently involves generating packages for multiple resolutions (720p and 1080p) by modifying `appinfo.json` with appropriate resolution and version.
    *   **Tizen**: Involves copying templates, modifying `config.xml` for version, and in PRO environments, interacting with the user to select a security profile for signing.
*   **Error Handling**: Consistent use of `console.error` and `process.exit(1)` for critical errors (e.g., invalid arguments, missing templates, no generated packages).
*   **App Configuration (`src\App.jsx`, `src\utils\platformUtils.js`)**:
    *   **Platform Detection**: The logic for determining the current platform (`window.PlatformUtils.getPlatform()`) is frequently adjusted, including handling user agent strings and query string overrides.
    *   **Keyboard Navigation**: Standardized keyboard input handling for Smart TVs (`IinitKeyboardNavigation_SmartTV`) is critical, with specific key codes for various actions (e.g., "Back").
    *   **Global Variables**: Extensive use of `window.AppGlobal` and `window.PlatformUtils` for storing application state, actions, and utility functions.
*   **Template Structure (`build_templates`)**: The `README.md` file reflects changes in naming conventions and expected output structure, indicating ongoing refinement of the build system. The `App3Cat_` prefix was changed to `3cat_` for template folders and `readAppName` fallback.
*   **Consistency in Utility Functions**: Functions like `parseArgs`, `pad2`, `getTodayYYYYMMDD`, `readAppName`, `ensureDir`, `listPackages`, `pickNewOrLatest`, `copyRecursive`, `removeDirRecursive` remain largely stable throughout the `package-launcher-apps.cjs` changes, serving as core building blocks.
